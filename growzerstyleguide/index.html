Buttons

  • Primary

    • Preview:

    • Structure: 

      <a class="btn btn-primary">Submit</a>
<a class="btn btn-primary disabled">Submit</a> <!-- The faded orange one, with no hover/pointer events -->
<a class="btn btn-primary disabled very">Submit</a> <!-- The faded gray one, again with no hover/pointer events -->
      Note the need for the 'btn' class as well. Promise to get this to a more uniform state sometime in the near future.

  • Secondary

    • Preview:

    • Structure: 

      <a class="btn-secondary-new">Print</a>
<a class="btn-secondary-new disabled">Print</a> <!-- The faded orange one, with no hover/pointer events -->
<a class="btn btn-secondary">Submit</a> <!-- THIS IS DEPRECATED! Don't use it. If you find it anywhere, it's only because of 
visual compatibility with pages that still use less-than-optimal-height text inputs. -->

Headers

  • Primary

    The topmost gray/white bar that's present in (almost) every page.

    • Preview:

    • Structure: 

      <div class="theme-header">
<!-- Both of these will have their content center-aligned -->
    <div></div> <!-- This will automatically be left-justified -->
    <div></div> <!-- This will automatically be right-justified -->
</div>
      I've opted for this strucure since I noticed the left-content/right-content separation is very commonly used.

  • Secondary

    The white bar present in some pages, usually in detail pages, that usually exposes additional actions such as filters or printing.

    • Preview:

    • Structure: 

      <div class="theme-header white">
<!-- Both of these will have their content center-aligned -->
    <div></div> <!-- This will automatically be left-justified -->
    <div></div> <!-- This will automatically be right-justified -->
</div>
